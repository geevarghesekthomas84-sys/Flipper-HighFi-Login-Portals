# Contributing to Flipper-HighFi-Login-Portals

Thank you for your interest in contributing to this research project! High-fidelity replicas are a vital part of UI/UX study and security awareness training.

## 🛡️ Ethics First

All contributions must adhere to our **Ethics & Legal Notice**. This project is strictly for authorized security research and educational purposes. We do not accept contributions that:
- Include real phishing infrastructure.
- Encourage illegal activities.
- Include malicious scripts or tracking beyond simple demo webhooks.

## 🚀 How to Contribute

### 🎨 Adding New Portals
If you have a high-fidelity replica of a login portal:
1.  **Self-Containment**: Ensure all CSS, JS, and SVG assets are inlined. The `index.html` should work offline without any external CDNs.
2.  **Responsiveness**: Test the UI on both desktop and mobile viewports.
3.  **Portability**: It must be under 500KB to ensure compatibility with ESP32-based modules.
4.  **Standard vs Discord**: Please provide both a `_standard` version (logs to local console/endpoint) and a `_discord` version (uses clear placeholder for webhook).

### 🐛 Reporting Bugs
Use our **Bug Report** template to report layout issues, broken links, or logic flaws in the templates.

### 💡 Suggestions
Open an issue with the `enhancement` label if you have ideas for new features (e.g., local storage persistence patterns).

## 📝 Style Guidelines
- Use Semantic HTML5.
- Maintain premium design aesthetics (smooth transitions, modern typography).
- Keep code clean and well-commented for other researchers.
